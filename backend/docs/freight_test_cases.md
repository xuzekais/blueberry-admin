# 蓝莓订单运费计算测试用例与规则

本文档记录运费结算的核心业务逻辑测试用例，后续每次修改运费相关代码时，必须保证以下测试用例全部通过。

## 场景配置设定 (Mock 模版数据)
假设运费模版配置了以下三条规则：
- **规则1 (省内普通果)**: 地区: [广东省内], 果径: [大果, 中果, 小果, 12+, 15+, 18+, 20+, 22+], 包邮条件: 满 1kg 包邮, 首重: 1kg/8元, 续重: 1kg/2元
- **规则2 (省内特殊果)**: 地区: [广东省内], 果径: [红果, 次果], 包邮条件: 0 (不包邮), 首重: 1kg/10元, 续重: 1kg/5元
- **规则3 (省外所有果)**: 地区: [广东省外], 果径: [全部], 包邮条件: 0 (不包邮), 首重: 1kg/15元, 续重: 1kg/6元

---

## 测试用例列表

### 用例 1：省内混合触发整单包邮
**购买商品**：1斤大果(0.5kg) + 1斤中果(0.5kg) + 1斤红果(0.5kg)
**收货地区**：广东省内
**业务预期**：大果与中果合并重量为 `0.5 + 0.5 = 1kg`，触发了“规则1”的 `满1kg包邮`。由于其他混合果已经满足包邮条件，附带的红果不需要单独计费。
**预期运费**：0 元

### 用例 2：省外不包邮统一计价
**购买商品**：1斤大果(0.5kg) + 1斤中果(0.5kg) + 1斤红果(0.5kg)
**收货地区**：广东省外
**业务预期**：因为规则3中包含了“全部”果径。所有物品总重量 `0.5 + 0.5 + 0.5 = 1.5kg`，落入“省外不包邮”逻辑。
计算过程：首重 15元(覆盖前1kg)，超出的 0.5kg 触发 1 次续重收费 (6元)。
**预期运费**：15 + 6 = 21 元

### 用例 3：省内未达包邮线，叠加收费
**购买商品**：0.5斤大果(0.25kg) + 1斤红果(0.5kg)
**收货地区**：广东省内
**业务预期**：大果未达到规则1的包邮线(1kg)，不能免去红果的邮费。两者分别独立按照各自阶梯计算后相加。
计算过程：大果 0.25kg 在规则1首重内 -> 8元。 红果 0.5kg 在规则2首重内 -> 10元。
**预期运费**：8 + 10 = 18 元
